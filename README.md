# Лабораторная работа №2 — Модули, пакеты и ООП в Python

## Описание проекта
Данный проект представляет собой MVP-приложение, демонстрирующее навыки:
- Модульной структуры Python-приложений
- Объектно-ориентированного программирования (ООП): наследование, абстрактные классы, свойства (@property)
- Применения декораторов для валидации и логирования
- Работы с локальной базой SQLite, а также экспорта данных в Word (.docx) и Excel (.xlsx)

Приложение моделирует работу виртуального музея:
- Пользователь оценивает экспонаты и статьи от 0 до 10
- Рассчитываются средние оценки для каждого объекта
- Объекты сортируются по среднему баллу
- Данные сохраняются в базу SQLite и экспортируются в отчёты

## Установка и запуск

### Вариант 1 — без Docker
```bash
git clone https://github.com/ApathyOn/lab-work-number-two.git
cd lab-work-number-two
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python main.py
```
После работы программы появятся файлы:

* `museum_summary_report.docx`
* `museum_summary_report.xlsx`
* `museum.db`

### Вариант 2 — через Docker

```bash
sudo docker build -t museum-app .
sudo docker run -it --rm -v $(pwd):/app museum-app
```
Отчёты и база данных сохраняются в текущей директории.

## Тестирование

Используется `pytest`. Проверяются:

* структура базы данных
* правильность пересчёта среднего рейтинга
* обработка ошибок при некорректных данных
* сортировка объектов по убыванию среднего рейтинга
* создание отчетов Word и Excel

Запуск тестов:

```bash
pytest -v
```
